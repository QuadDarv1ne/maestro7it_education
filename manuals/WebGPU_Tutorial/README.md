# WebGPU: Полное руководство по обработке изображений

## Введение
WebGPU — это современный графический API, пришедший на смену WebGL. Его главная особенность — прямой доступ к **Compute Shaders (вычислительным шейдерам)**. Это позволяет использовать видеокарту для параллельных вычислений, что идеально подходит для обработки изображений.

## Часть 1: Теория

### Почему GPU быстрее CPU?
Представьте изображение Full HD (1920x1080). Это более 2 миллионов пикселей.
*   **CPU** обрабатывает их последовательно (или небольшими потоками). Это медленно.
*   **GPU** имеет тысячи ядер и обрабатывает тысячи пикселей **одновременно**.

### Архитектура WebGPU
1.  **Device (Устройство):** Логическое соединение с видеокартой.
2.  **Pipeline (Конвейер):** Описание того, как данные будут обрабатываться (содержит шейдер).
3.  **Bind Group (Группа привязки):** "Клей", соединяющий данные (текстуры) и шейдер.
4.  **Command Buffer (Буфер команд):** Список инструкций, который JS отправляет в GPU.

## Часть 2: Практика (Grayscale)

Мы реализуем фильтр "Оттенки серого".

### Шаг 1: WGSL Шейдер
Код шейдера находится в файле `shader.wgsl`.
Суть: мы запускаем функцию `main` для каждого пикселя одновременно. 
Функция `textureLoad` читает цвет, мы считаем яркость по формуле `(0.299*R + 0.587*G + 0.114*B)`, и `textureStore` записывает результат.

### Шаг 2: JavaScript
Логика работы (файл `script.js`):
1.  Инициализация `navigator.gpu.requestAdapter()`.
2.  Создание текстур: Input (откуда читаем) и Output (куда пишем).
3.  Создание Pipeline и Bind Group.
4.  Команда `dispatchWorkgroups(width/8, height/8)` — запуск вычислений.

## Часть 3: Важные нюансы

### 1. Работа с памятью (Usage Flags)
WebGPU строг к памяти. Текстура должна быть создана с правильными флагами:
*   `COPY_DST`: В нее можно копировать данные.
*   `STORAGE_BINDING`: В нее можно писать из шейдера.
*   `TEXTURE_BINDING`: Из нее можно читать в шейдере.

### 2. Race Condition (Состояние гонки)
**Никогда не читайте и не пишите в одну и ту же текстуру в одном шейдере!**
GPU обрабатывает пиксели в случайном порядке. Если вы читаете и пишете в одну память, результат будет хаотичным и испорченным. Всегда используйте пары: Input Texture -> Output Texture.

### 3. Чтение результата (Readback)
Чтобы увидеть результат на странице, нужно:
1.  Создать буфер `GPUBuffer` с флагом `MAP_READ`.
2.  Скопировать в него данные из Output Texture (`copyTextureToBuffer`).
3.  "Замапить" буфер (`buffer.mapAsync`) и прочитать сырые байты.
Это самая медленная часть процесса, поэтому делать это нужно только в конце.

## Часть 4: Запуск примера
1.  Откройте `index.html` через локальный сервер (например, Live Server в VS Code или `python -m http.server`).
2.  Откройте консоль браузера (F12) для просмотра логов.
3.  Веб-камера запросит разрешение, после чего вы увидите видео, обработанное шейдером в реальном времени.
