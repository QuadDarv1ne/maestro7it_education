## 1. Введение

### 1.1 Классификация программного обеспечения

Программы для RTL-SDR можно разделить на несколько категорий по их назначению:

| Категория | Назначение | Примеры |
|-----------|------------|---------|
| **Универсальные приёмники** | Панорамный обзор, приём аналоговых сигналов | SDR#, GQRX, SDRangel |
| **Специализированные декодеры** | Декодирование конкретных протоколов | dump1090, WXtoImg, aisdeco2 |
| **Анализаторы** | Измерение параметров сигнала | rtl_power, Spektrum |
| **Серверы** | Удалённый доступ к SDR | rtl_tcp, SpyServer |
| **Мобильные** | Работа на смартфонах | SDR Touch, RF Analyzer |

### 1.2 Требования к системе

**Минимальные требования:**
- Процессор: 2 ядра, 2+ ГГц
- ОЗУ: 2 ГБ (4 ГБ рекомендуется)
- USB 2.0 или выше
- ОС: Windows 7+, Linux, macOS 10.12+

**Рекомендуемые требования для многозадачности:**
- Процессор: 4+ ядер
- ОЗУ: 8+ ГБ
- USB 3.0

### 1.3 Виртуальные аудиокабели

Многие программы работают в связке: SDR-приёмник выводит аудио, которое декодер принимает и обрабатывает. Для этого нужны виртуальные аудиокабели:

| Платформа | Программа | Описание |
|-----------|-----------|----------|
| Windows | **VB-Cable** (vb-audio.com) | Бесплатный, простой |
| Windows | **Virtual Audio Cable** | Платный, больше функций |
| Linux | **PulseAudio** | Встроен в систему |
| Linux | **JACK** | Профессиональный, сложнее |
| macOS | **BlackHole** | Бесплатный |
| macOS | **Loopback** | Платный, мощный |

## 2. Универсальные SDR-приёмники

### 2.1 SDR# (SDRSharp)

**Платформа:** Windows  
**Цена:** Бесплатно  
**Сайт:** airspy.com/download  
**Лучший для:** Начинающих, общий сканирование

#### Описание

`SDR#` — самая популярная программа для RTL-SDR на Windows. Разработана командой Airspy, отличается простым интерфейсом и богатой экосистемой плагинов.

Это идеальная отправная точка для новичков благодаря интуитивному дизайну и обширной документации.

Программа обеспечивает плавное отображение спектра в реальном времени и поддерживает практически все виды модуляции, с которыми может столкнуться начинающий радиолюбитель.

#### Возможности

- Панорамный спектр и водопад в реальном времени
- Поддержка всех типов модуляции: WFM, NFM, AM, USB, LSB, CW, DSB
- Запись аудио и IQ-данных
- Частотные закладки и память
- Плагины для расширения функционала
- Поддержка множества SDR-устройств

#### Установка

1. Скачайте `SDR#` с официального сайта `Airspy`. Программа распространяется в виде ZIP-архива, который не требует установки — достаточно распаковать его в любую папку на вашем компьютере.

2. Убедитесь, что драйверы RTL-SDR уже установлены через Zadig (см. раздел про драйверы).

3. Запустите SDRSharp.exe из папки программы.

4. В выпадающем списке Source (вверху слева) выберите RTL-SDR USB. Если устройство не появляется, проверьте, что драйвер установлен корректно и RTL-SDR подключён к USB-порту.

5. Нажмите кнопку Play (синий треугольник) для начала приёма.

#### Базовая настройка

**Настройки RTL-SDR (кнопка с шестерёнкой):**

| Параметр | Рекомендуемое значение | Пояснение |
|----------|------------------------|-----------|
| RF Gain | 30–40 dB | Подберите по ситуации |
| Sample Rate | 2.048 MSPS | Стабильное значение |
| Frequency Correction (PPM) | 0–2 | Для V4 обычно 0 |
| Direct Sampling | Quadrature Sampling | Для HF/КВ-диапазона |

**Настройки модуляции (выпадающий список):**

| Модуляция | Применение |
|-----------|------------|
| WFM | FM-радио (88–108 МГц) |
| NFM | Узкополосная FM: служебные, морские, радиолюбители |
| AM | Авиация (118–137 МГц), AM-вещание |
| USB | SSB верхняя боковая: радиолюбители выше 10 МГц |
| LSB | SSB нижняя боковая: радиолюбители ниже 10 МГц |
| CW | Азбука Морзе |
| DSB | Двухполосная AM (редко используется) |

#### Популярные плагины

**Frequency Manager + Scanner**

Мощный сканер частот с функцией сохранения и организации частот в базы данных. Позволяет создавать списки частот по категориям, автоматически сканировать диапазоны и останавливаться при обнаружении сигнала.

**TETRA Demodulator**

Декодер цифровой связи TETRA, используемой в Европе службами безопасности и предприятиями. Позволяет прослушивать незашифрованные переговоры.

**SDR# Noise Reduction**

Продвинутый шумоподавитель, улучшающий качество приёма слабых сигналов. Особенно полезен на КВ-диапазонах с высоким уровнем атмосферных и индустриальных помех.

**Level Meter**

Измеритель уровня сигнала с возможностью логирования. Полезен для анализа условий прохождения радиоволн и сравнения эффективности антенн.

#### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| Пробел | Старт/стоп приёма |
| F | Ввод частоты |
| G | Настройка Gain |
| M | Выбор модуляции |
| B | Добавить закладку |
| Стрелки | Перемещение по частоте |

### 2.2 GQRX SDR

**Платформа:** Linux, macOS  
**Цена:** Бесплатно (open-source)  
**Сайт:** gqrx.dk  
**Лучший для:** Пользователей Linux/macOS

#### Описание

GQRX — основной выбор для пользователей Linux и macOS. Это полностью открытая программа, построенная на библиотеке GNU Radio, которая предоставляет профессиональные возможности для анализа радиосигналов. Программа активно развивается сообществом и поддерживает широкий спектр SDR-устройств благодаря архитектуре на основе SoapySDR — универсального интерфейса для различных приёмников.

#### Возможности

- Приём и демодуляция различных сигналов
- Спектральный анализ и водопад
- Запись IQ-данных и аудио
- Встроенный FFT-анализатор
- Поддержка сетевых SDR через SoapyRemote
- Интеграция с GNU Radio

#### Установка

**Ubuntu/Debian:**

```bash
sudo apt update
sudo apt install gqrx-sdr
```

**Fedora:**

```bash
sudo dnf install gqrx
```

**macOS (через Homebrew):**

```bash
brew install --cask gqrx
```

**Из исходников:**

```bash
git clone https://github.com/gqrx-sdr/gqrx.git
cd gqrx
mkdir build
cd build
cmake ..
make
sudo make install
```

#### Особенности интерфейса

GQRX имеет более «инженерный» интерфейс по сравнению с SDR#. Главное окно разделено на три области: спектр (сверху), водопад (посередине) и приёмник (снизу). Каждая область настраивается отдельно.

**Важные настройки:**

1. **FFT Size** — размер БПФ, влияет на разрешение по частоте. Большие значения (32768+) дают лучшее разрешение, но требуют больше процессорного времени.

2. **FFT Window** — оконная функция для FFT. Blackman-Harris — хороший выбор для общего использования.

3. **PAN adapter** — режим отображения спектра, популярный среди радиолюбителей.

#### Настройка для RTL-SDR V4

При первом запуске GQRX предложит настроить устройство. Для RTL-SDR V4:

1. Выберите Device: RTL-SDR
2. Input rate: 2000000 (2 MSPS) — стабильное значение
3. Для HF-диапазона включите Direct Sampling в настройках

### 2.3 SDRangel

**Платформа:** Windows, Linux, macOS  
**Цена:** Бесплатно (open-source)  
**Сайт:** github.com/f4exb/sdrangel  
**Лучший для:** Продвинутых пользователей, мультиканальный приём

#### Описание

SDRangel — мощная профессиональная программа с поддержкой множества каналов приёма одновременно. Это означает, что вы можете слушать несколько частот в разных модуляциях параллельно, используя одно устройство. Программа имеет встроенные декодеры для многих цифровых протоколов, что делает её универсальным решением для серьёзных энтузиастов.

#### Ключевые возможности

- **Мультиканальный приём** — несколько независимых приёмников одновременно
- Встроенные декодеры ADS-B, AIS, DAB, DMR, TETRA, NXDN, POC12/16/32
- Поддержка передачи (TX) для совместимых устройств
- Анализатор спутниковых сигналов
-Map-интеграция для ADS-B и AIS
- Запись и воспроизведение IQ-данных

#### Установка

**Windows:**

Скачайте установщик с GitHub Releases: github.com/f4exb/sdrangel/releases

**Linux:**

```bash
# Ubuntu/Debian
sudo add-apt-repository ppa:ed10vi86/sdrangel
sudo apt update
sudo apt install sdrangel

# Flatpak
flatpak install flathub org.sdrangel.SDRangel
```

**macOS:**

```bash
brew install --cask sdrangel
```

#### Особенности работы

SDRangel использует концепцию «каналов» — каждый канал представляет отдельный приёмник или декодер. Вы можете добавить несколько каналов в пределах полосы пропускания вашего SDR.

**Добавление канала:**

1. Нажмите кнопку «Add channel»
2. Выберите тип канала (например, NFM Demodulator)
3. Настройте частоту в пределах видимого спектра

**Пример мультиканальной настройки:**

| Канал | Тип | Частота | Назначение |
|-------|-----|---------|------------|
| 1 | ADS-B Demodulator | 1090 MHz | Отслеживание самолётов |
| 2 | NFM Demodulator | 118.100 MHz | Авиационная вышка |
| 3 | NFM Demodulator | 121.500 MHz | Аварийная частота |

## 3. Авиационное ПО

### 3.1 dump1090

**Платформа:** Windows, Linux, macOS  
**Цена:** Бесплатно (open-source)  
**Сайт:** github.com/flightaware/dump1090  
**Лучший для:** ADS-B, отслеживание самолётов

#### Описание

dump1090 — стандарт де-факто для приёма ADS-B сигналов на 1090 МГц. Разработан FlightAware, одним из крупнейших сервисов отслеживания авиации. Программа декодирует сигналы трансpondеров самолётов и отображает их на карте в реальном времени, а также может отправлять данные на агрегаторы типа FlightAware.

#### Принцип работы

ADS-B (Automatic Dependent Surveillance-Broadcast) — система, при которой самолёты автоматически транслируют свои данные:

- Идентификатор (ICAO hex code, номер рейса)
- Координаты (широта, долгота)
- Высоту (барометрическая и GNSS)
- Путевую скорость и скорость относительно земли
- Курс и вертикальную скорость
- Тип воздушного судна

dump1090 принимает эти сигналы на 1090 МГц и декодирует их в режиме реального времени.

#### Установка

**Windows:**

1. Скачайте релиз с GitHub: github.com/flightaware/dump1090/releases
2. Распакуйте архив в папку
3. Убедитесь, что драйверы RTL-SDR установлены

**Linux:**

```bash
# Ubuntu/Debian
sudo apt install dump1090-mutability

# Или из исходников
git clone https://github.com/flightaware/dump1090
cd dump1090
make
```

#### Запуск

**Базовый запуск:**

```bash
dump1090 --interactive --net
```

**Основные параметры:**

| Параметр | Описание |
|----------|----------|
| `--interactive` | Интерактивный режим с таблицей самолётов |
| `--net` | Включить веб-интерфейс на порту 8080 |
| `--net-http-port 8080` | Порт веб-интерфейса |
| `--gain 40` | Усиление в dB |
| `--ppm 1` | Коррекция частоты PPM |
| `--aggressive` | Агрессивное декодирование (больше сообщений, но возможны ошибки) |
| `--stats` | Показывать статистику |
| `--quiet` | Минимальный вывод в консоль |

**Полный пример:**

```bash
dump1090 --interactive --net --gain 45 --ppm 0 --stats-every 60
```

#### Веб-интерфейс

После запуска с параметром `--net` откройте браузер:

http://localhost:8080

Вы увидите карту с самолётами в радиусе приёма (обычно 200–400 км при хорошей антенне).

#### Интеграция с FlightAware

Вы можете отправлять данные на FlightAware и получать премиум-аккаунт бесплатно:

1. Зарегистрируйтесь на flightaware.com
2. Создайте feeder-аккаунт
3. Используйте piaware для отправки данных

#### Альтернативные версии

**dump1090-fa** — версия от FlightAware с улучшениями

**dump1090-mutability** — популярная форк-версия с дополнительными функциями

**readsb** — современная высокопроизводительная альтернатива

### 3.2 RTL1090

**Платформа:** Windows  
**Цена:** Бесплатно  
**Сайт:** jetvision.de  
**Лучший для:** ADS-B на Windows, простой интерфейс

#### Описание

RTL1090 — Windows-программа для ADS-B с графическим интерфейсом. Проще в использовании, чем dump1090 для начинающих пользователей Windows, не требует работы с командной строкой.

#### Особенности

- Графический интерфейс Windows
- Таблица самолётов в реальном времени
- Экспорт данных в различные форматы
- Интеграция с Virtual Radar Server
- Звуковые оповещения

#### Использование

1. Установите драйверы RTL-SDR через Zadig
2. Запустите RTL1090.exe
3. Нажмите Start
4. Самолёты появятся в таблице

### 3.3 Virtual Radar Server

**Платформа:** Windows  
**Цена:** Бесплатно  
**Сайт:** virtualradarserver.co.uk  
**Лучший для:** Визуализация ADS-B, база данных самолётов

#### Описание

Virtual Radar Server (VRS) — программа для визуализации ADS-B данных с богатой базой информации о самолётах. Может работать как самостоятельный приёмник или как клиент для dump1090/RTL1090.

#### Возможности

- Карта самолётов в веб-интерфейсе
- База данных типов самолётов и маршрутов
- История полётов
- Плагины и расширения
- Мультиязычный интерфейс

#### Настройка с dump1090

1. Запустите dump1090 с параметром `--net`
2. Запустите Virtual Radar Server
3. В настройках добавьте источник: localhost, порт 30003

### 3.4 acarsdec

**Платформа:** Windows, Linux  
**Цена:** Бесплатно (open-source)  
**Сайт:** github.com/TLeconte/acarsdec  
**Лучший для:** ACARS, текстовые сообщения самолётов

#### Описание

acarsdec декодирует ACARS (Aircraft Communications Addressing and Reporting System) — систему текстовых сообщений между самолётами и наземными станциями. ACARS работает на частоте 131.550 МГц (основная) и содержит служебные сообщения о рейсах.

#### Что можно принять

- Время вылета/прилёта (OOOI — Out, Off, On, In)
- Метеорологические данные
- Технические сообщения
- Вольные сообщения экипажа

#### Установка и запуск

**Linux:**

```bash
git clone https://github.com/TLeconte/acarsdec
cd acarsdec
mkdir build
cd build
cmake ..
make
```

**Запуск:**

```bash
acarsdec -r 0 131550000 131725000
```

### 4.1 WXtoImg

Спутники передают изображения Земли в формате APT (Automatic Picture Transmission) на частотах около 137 МГц. WXtoImg преобразует аналоговый сигнал в полноценные фотографии с geographical overlays.

#### Поддерживаемые спутники

| Спутник | Частота | Статус |
|---------|---------|--------|
| NOAA 15 | 137.620 МГц | Активен |
| NOAA 18 | 137.9125 МГц | Активен |
| NOAA 19 | 137.100 МГц | Активен |

#### Установка

**Windows:**

Скачайте установщик с wxtoimg.com (бесплатная версия в архиве)

**Linux:**

```bash
# Загрузите .deb или .rpm пакет с сайта
sudo dpkg -i wxtoimg_*.deb
```

#### Подготовка к приёму

**Антенна:**

Для приёма NOAA нужна специальная антенна с круговой поляризацией:

- **QFH (Quadrifilar Helix)** — лучшая антенна для NOAA
- **Turnstile** — простая альтернатива
- **Двойной квадрат** — можно сделать самому

**Длина стороны квадрата для 137 МГц:** ≈ 54 см

#### Процесс приёма

**Шаг 1: Определение времени пролёта**

Используйте программу GPredict или сайт n2yo.com для определения времени пролёта спутника над вашей местностью. Пролёты происходят несколько раз в день, но качественные изображения получаются при высоте спутника над горизонтом более 30–40 градусов.

**Шаг 2: Настройка SDR#**

1. Настройтесь на частоту спутника (например, 137.620 МГц для NOAA 15)
2. Выберите модуляцию WFM
3. Полоса пропускания: ~40 кГц
4. Включите запись аудио (WAV)

**Шаг 3: Учёт доплеровского сдвига**

Частота сигнала меняется из-за движения спутника. При приближении частота выше, при удалении — ниже. Сдвиг может достигать ±3–4 кГц.

Варианты учёта:
- Ручная подстройка во время пролёта
- Использование GPredict для автоматической подстройки через DDE-плагин для SDR#
- Запись с запасом по полосе

**Шаг 4: Декодирование**

1. Откройте WXtoImg
2. File → Open Audio File
3. Выберите записанный WAV-файл
4. Программа автоматически декодирует изображение

#### Обработка изображений

WXtoImg может создавать различные варианты изображений:

| Тип | Описание |
|-----|----------|
| **Thermal** | Тепловое изображение (инфракрасный канал) |
| **Visible** | Видимый свет |
| **Enhanced** | Улучшенное цветное изображение |
| **MSA** | Multispectral Analysis |
| **HVCT** | High-resolution Vegetation/Cloud/Temperature |

#### Полезные настройки WXtoImg

**Options → Recording Options:**

- Установите источник звука (ваш SDR или виртуальный кабель)
- Настройте автоматическую запись при пролёте

**Options → Satellite Pass List:**

- Введите ваши координаты
- Программа покажет предстоящие пролёты

**Options → Map Overlay:**

- Добавьте карту для наложения на изображение
- Показывает границы стран, города

```bash
rtl_power -f 88M:108M:10k -i 10 -g 40 fm_scan.csv
```

Параметры:
- `-f 88M:108M:10k` — диапазон от 88 до 108 MHz с шагом 10 kHz
- `-i 10` — интеграция 10 секунд
- `-g 40` — gain 40 dB
- `fm_scan.csv` — выходной файл

**Сканирование всего диапазона RTL-SDR:**

```bash
rtl_power -f 24M:1766M:100k -i 15 -g 30 full_scan.csv
```

#### Визуализация

Используйте Python-скрипт rtl_power.py или Heatmap для визуализации:

```bash
python rtl_power.py fm_scan.csv fm_scan.png
```

## 10. Драйверы и утилиты

### 10.1 Zadig

**Платформа:** Windows  
**Цена:** Бесплатно  
**Сайт:** zadig.akeo.ie  
**Лучший для:** Установка драйверов RTL-SDR

#### Описание

Zadig — утилита для установки драйверов USB-устройств на Windows. Критически важна для RTL-SDR, так как стандартные драйверы Windows не подходят для SDR-режима.

#### Использование

1. Подключите RTL-SDR к USB
2. Запустите Zadig от имени администратора
3. Options → List All Devices
4. Найдите устройство (обычно «Bulk-In, Interface (Interface 0)» или «RTL2832UHIDIR»)
5. Выберите драйвер: **WinUSB**
6. Нажмите «Replace Driver» или «Install Driver»

**Важно:** Для V4 драйвер может определяться иначе, чем для V3. Если видите несколько устройств с RTL в названии, попробуйте установить драйвер для каждого.

### 10.2 rtl-sdr библиотека

**Платформа:** Linux, macOS  
**Цена:** Бесплатно (open-source)  
**Сайт:** osmocom.org/projects/rtl-sdr/wiki  

#### Описание

Базовая библиотека и утилиты для RTL-SDR на Linux/macOS. Включает rtl_tcp, rtl_test, rtl_fm, rtl_power.

#### Установка

```bash
# Ubuntu/Debian
sudo apt install rtl-sdr

# Из исходников
git clone https://git.osmocom.org/rtl-sdr
cd rtl-sdr
mkdir build
cd build
cmake ../ -DINSTALL_UDEV_RULES=ON
make
sudo make install
sudo ldconfig
```

#### Утилиты

**rtl_test — тестирование устройства:**

```bash
rtl_test -t
```

Показывает информацию о тюнере и проверяет работоспособность.

**rtl_fm — простой FM-приёмник:**

```bash
rtl_fm -f 100.0M -M wfm -g 40 | play -r 32k -t raw -e s -b 16 -c 1 -V1 -
```

**rtl_tcp — сетевой сервер:**

```bash
rtl_tcp -a 0.0.0.0 -p 1234
```

Позволяет использовать RTL-SDR по сети. Клиенты подключаются к IP:порт.
