# Полный справочник платформ для развёртывания проектов (2025-2026)

Этот документ содержит подробное описание актуальных платформ для публикации и хостинга веб-приложений, API, ботов и других IT-проектов. Каждая платформа описана с учётом её особенностей, тарифов, преимуществ и недостатков.

---

## Быстрый выбор

### По типу проекта

| Что нужно? | Платформа | Почему |
|------------|-----------|--------|
| Telegram-бот | Railway / Amvera | Просто, есть БД, недорого |
| Next.js сайт | Vercel | Нативная поддержка, edge functions |
| Статический сайт | Netlify / Vercel | Бесплатно, CDN, формы |
| API с PostgreSQL | Supabase / Render | Готовая БД, REST API |
| High-load глобально | Cloudflare / Fly.io | Edge-сеть, низкая latency |
| 152-ФЗ (РФ) | Amvera / Timeweb | Локализация данных |
| Максимально дёшево | Coolify на VPS | ~$5/мес за полный контроль |
| Full-stack приложение | Render / Railway / Koyeb | Бэкенд + фронтенд + БД в одном месте |

### По типу поддержки

| Тип | Платформы |

| **Frontend** | Vercel, Netlify, Cloudflare Pages |

| **Full-stack** | Render, DigitalOcean App Platform, Amvera, Timeweb Cloud, Northflank |
| **Self-hosted** | Coolify, CapRover |

## Международные PaaS-платформы

### Render

![Type](https://img.shields.io/badge/type-Full--stack-blue)
![Free Tier](https://img.shields.io/badge/free%20tier-yes-green)
![Docker](https://img.shields.io/badge/docker-supported-blue)

**Официальный сайт:** https://render.com

#### Общее описание

Render — это современная облачная платформа для разработчиков, которая позиционируется как «облако для строителей» (The cloud for builders). Платформа позволяет развёртывать и масштабировать любые приложения — от первого пользователя до миллиардного. Render обеспечивает интуитивную облачную инфраструктуру для современного веба без необходимости управления DevOps-процессами. В январе 2025 года компания привлекла $80 миллионов в рамках Series C финансирования, что подтверждает её устойчивость и популярность среди разработчиков.

#### Ключевые возможности

- **Автоматическое развёртывание из Git** — подключение к GitHub или GitLab с автоматическим деплоем при каждом push
- **Поддержка контейнеров Docker** — возможность деплоя собственных Docker-образов
- **Managed базы данных** — PostgreSQL, MySQL, Redis с автоматическими бэкапами
- **Автомасштабирование** — горизонтальное и вертикальное масштабирование по запросу
- **SSL-сертификаты** — автоматическое получение и обновление Let's Encrypt
- **Preview environments** — изолированные окружения для каждой pull request
- **Static sites** — бесплатный хостинг статических сайтов
- **Background workers** — фоновые задачи и очереди
- **Cron jobs** — планировщик задач

#### Тарифы и ценообразование

| План | Стоимость | Особенности |
|------|-----------|-------------|
| Free | $0/мес | Статические сайты, 750 часов/мес для сервисов, база данных PostgreSQL на 90 дней |
| Individual | $19/мес | Базовый план для личных проектов + оплата использованных ресурсов |
| Team | $19/мес на пользователя | Командная работа, расширенные функции |

**Примеры стоимости вычислительных ресурсов:**
- 2GB RAM, 1 CPU — ~$25/мес
- 4GB RAM, 2 CPU — ~$50/мес

#### Преимущества

- Отличный баланс между простотой использования и мощностью
- Щедрый бесплатный тариф для статических сайтов
- Понятная модель ценообразования
- Автоматические SSL-сертификаты
- Хорошая документация и поддержка
- Preview-окружения для тестирования

#### Недостатки

- Нет постоянного бесплатного тарифа для баз данных (PostgreSQL доступен бесплатно только 90 дней)
- Базовая плата $19/мес за workspace даже при минимальном использовании
- Ограниченные регионы развёртывания
- Может быть дороже конкурентов при интенсивном использовании

#### Идеально подходит для

- Full-stack веб-приложений (React, Vue, Node.js, Python, Go, Rust)
- API-серверов и микросервисов
- Статических сайтов и JAMstack-проектов
- Команд, которым нужен простой деплой без DevOps

### Railway

![Type](https://img.shields.io/badge/type-Backend-green)
![Free Tier](https://img.shields.io/badge/free%20tier-yes-green)
![Docker](https://img.shields.io/badge/docker-supported-blue)

**Официальный сайт:** https://railway.com

#### Общее описание

Railway — это универсальная облачная платформа «всё-в-одном», которая позволяет разработчикам быстро развёртывать приложения с минимальной конфигурацией. Платформа отличается простотой использования и посекундной оплатой только за фактически использованные ресурсы. Railway автоматически определяет конфигурацию проекта при наличии Dockerfile, что делает процесс деплоя максимально простым.

#### Ключевые возможности

- **Автоопределение конфигурации** — автоматический деплой при наличии Dockerfile
- **PostgreSQL, MySQL, Redis, MongoDB** — managed базы данных одним кликом
- **Переменные окружения** — удобное управление секретами и конфигурацией
- **Мониторинг и логи** — встроенная панель с метриками и логами в реальном времени
- **Развёртывание из Git** — интеграция с GitHub и GitLab
- **Docker-образы** — поддержка деплоя из Docker Registry
- **Volumes** — постоянные тома для хранения данных
- **TCP и UDP порты** — поддержка любых протоколов

#### Тарифы и ценообразование

| План | Стоимость | Особенности |
|------|-----------|-------------|
| Free | $0/мес | $1 бесплатного кредита ежемесячно для тестирования |
| Hobby | от $5/мес | Для личных проектов indie-разработчиков |
| Pro | от $20/мес | Командная работа, расширенные функции |

**Модель ценообразования:**
- Оплата посекундно только за использованные ресурсы
- Небольшое Node.js приложение может стоить $2-5/мес
- Высоконагруженное приложение — пропорционально потреблению

#### Преимущества

- Очень простое начало работы
- Посекундная оплата — платите только за то, что используете
- Быстрое развёртывание баз данных
- Отличный интерфейс и UX
- Щедрый начальный кредит $5 для новых пользователей
- Невыключение сервисов при низком потреблении

#### Недостатки

- После исчерпания начального кредита сервисы останавливаются до пополнения
- Меньше регионов по сравнению с конкурентами
- Ограниченные возможности мониторинга
- Нет встроенного CDN

#### Идеально подходит для

- Быстрых прототипов и MVP
- Персональных проектов indie-разработчиков
- Малых и средних приложений с переменной нагрузкой
- Telegram-ботов и небольших API

### Fly.io

![Type](https://img.shields.io/badge/type-Backend-green)
![Free Tier](https://img.shields.io/badge/free%20tier-limited-orange)
![Docker](https://img.shields.io/badge/docker-supported-blue)

**Официальный сайт:** https://fly.io

#### Общее описание

Fly.io — это уникальная облачная платформа, которая позволяет запускать приложения ближе к пользователям благодаря глобальной сети edge-локаций. Основная идея Fly.io — развернуть код там, где находятся пользователи, минимизируя задержки. В октябре 2024 года платформа перешла на новую модель ценообразования Pay As You Go, приблизившись к serverless-модели.

#### Ключевые возможности

- **Глобальная сеть** — развёртывание в 30+ регионах по всему миру
- **Fly Machines** — быстрые виртуальные машины, запускающиеся за секунды
- **Автомасштабирование** — scale-to-zero и автоматическое масштабирование
- **PostgreSQL и SQLite** — managed базы данных с репликацией
- **Edge caching** — кэширование на краю сети
- **Private networking** — защищённая сеть между сервисами
- **TCP/UDP** — поддержка любых протоколов
- **CLI и API** — полный контроль через командную строку

#### Тарифы и ценообразование

**Модель ценообразования (с октября 2024):**
- Оплата по факту использования ресурсов
- ~$5 за 30 дней за 1 GB дополнительной RAM
- CPU-пресеты с фиксированной ценой

**Примеры стоимости:**
- Небольшое приложение с PostgreSQL, Redis и storage — ~$50/мес
- Минимальный сервис — от $3-5/мес

#### Преимущества

- Уникальная edge-архитектура — приложения ближе к пользователям
- Быстрый запуск машин (секунды)
- Глобальное распределение нагрузки
- Scale-to-zero — не платите за простой
- Отличный CLI-инструмент
- Поддержка сложных архитектур

#### Недостатки

- Может быть дороже конкурентов при интенсивном использовании
- Более крутая кривая обучения
- Меньше готовых решений «из коробки»
- Требует понимания работы с контейнерами

#### Идеально подходит для

- Глобальных приложений с пользователями по всему миру
- Real-time приложений (игры, чаты, стриминг)
- Микросервисных архитектур
- Проектов, критичных к задержкам (latency)

### Vercel

![Type](https://img.shields.io/badge/type-Frontend-purple)
![Free Tier](https://img.shields.io/badge/free%20tier-yes-green)
![Docker](https://img.shields.io/badge/docker-limited-orange)

**Официальный сайт:** https://vercel.com

#### Общее описание

Vercel — это ведущая платформа для фронтенд-разработчиков, созданная авторами Next.js. Платформа оптимизирована для React, Next.js, Vue, Svelte и других современных фреймворков. Vercel обеспечивает бесшовную интеграцию с GitHub, GitLab и Bitbucket, предлагая лучший опыт для JAMstack и serverless-приложений.

#### Ключевые возможности

- **Нативная поддержка Next.js** — полная интеграция всех возможностей фреймворка
- **Edge Functions** — serverless-функции на краю сети
- **Automatic CI/CD** — деплой при каждом push в репозиторий
- **Preview Deployments** — уникальный URL для каждой ветки/PR
- **Analytics** — встроенная аналитика посещений
- **Edge Config** — конфигурация без передеплоя
- **Image Optimization** — автоматическая оптимизация изображений
- **Vercel AI SDK** — инструменты для AI-приложений

#### Тарифы и ценообразование

| План | Стоимость | Особенности |
|------|-----------|-------------|
| Hobby | $0/мес | Бесплатно навсегда для личных проектов, 100 GB bandwidth |
| Pro | $20/мес на пользователя | Командная работа, $20 кредитов на использование, расширенные лимиты |
| Enterprise | Договорная | Корпоративные функции, SLA, приоритетная поддержка |

**Дополнительные расходы:**
- Превышение bandwidth — от $40/ТБ
- Serverless function execution — по факту использования
- Edge requests — при превышении лимитов

#### Преимущества

- Лучший опыт для Next.js и React-проектов
- Молниеносный деплой и preview-окружения
- Бесплатный тариф для личных проектов
- Отличная интеграция с GitHub
- Edge-функции по всему миру
- Автоматическая оптимизация изображений
- AI SDK для современных AI-приложений

#### Недостатки

- Стоимость может быстро расти при превышении лимитов
- Ограниченная поддержка backend-сервисов
- Нет долгоживущих процессов (workers, cron)
- Привязка к экосистеме Vercel

#### Идеально подходит для

- Next.js и React-приложений
- JAMstack-сайтов и статических сайтов
- Landing pages и маркетинговых страниц
- Фронтенд-приложений с serverless API

### Netlify

![Docker](https://img.shields.io/badge/docker-limited-orange)

**Официальный сайт:** https://netlify.com

#### Общее описание

Netlify — это composable-веб-платформа, которая упрощает разработку, деплой и управление современными веб-проектами. Платформа специализируется на статических сайтах и JAMstack-архитектуре, предлагая мощные инструменты для контент-менеджмента и автоматизации CI/CD.

#### Ключевые возможности

- **Deploy from Git** — автоматический деплой из GitHub, GitLab, Bitbucket
- **Netlify Functions** — serverless-функции на AWS Lambda
- **Netlify Forms** — обработка форм без backend
- **Netlify Identity** — аутентификация пользователей
- **Split Testing** — A/B тестирование
- **Analytics** — аналитика без cookies
- **Edge Handlers** — обработка запросов на краю сети
- **Netlify CMS** — headless CMS для контента

#### Тарифы и ценообразование

| План | Стоимость | Особенности |
|------|-----------|-------------|
| Free | $0/мес | 125,000 запросов/мес, 100 GB bandwidth, HTTPS |
| Personal | $9/мес | Расширенные лимиты, приоритетная поддержка |
| Pro | $20/мес на пользователя | Командная работа, metered usage, расширенные функции |
| Business | Договорная | Корпоративные функции |

#### Преимущества

- Щедрый бесплатный тариф для статических сайтов
- Отличные инструменты для JAMstack
- Netlify Forms — формы без backend
- Netlify Identity — встроенная аутентификация
- Простота использования
- Интеграция с headless CMS

#### Недостатки

- Ограниченные возможности для backend
- Нет долгоживущих процессов
- Стоимость растёт при активном использовании
- Меньше регионов для serverless-функций

#### Идеально подходит для

- Статических сайтов и блогов
- JAMstack-проектов
- Landing pages
- Документации (Docusaurus, MkDocs)
- Портфолио и персональных сайтов

### Heroku

![Type](https://img.shields.io/badge/type-Backend-green)
![Free Tier](https://img.shields.io/badge/free%20tier-no-red)
![Docker](https://img.shields.io/badge/docker-supported-blue)

**Официальный сайт:** https://heroku.com

#### Общее описание

Heroku — одна из старейших и наиболее известных PaaS-платформ, принадлежащая Salesforce. Heroku стала эталоном для многих современных платформ деплоя и остаётся популярным выбором благодаря простоте использования и богатой экосистеме add-ons. Платформа поддерживает множество языков программирования и предлагает managed-базы данных.

#### Ключевые возможности

- **Dynos** — контейнеры для запуска приложений
- **Buildpacks** — автоматическое определение стека технологий
- **Heroku Postgres** — managed PostgreSQL с репликацией
- **Heroku Redis** — managed Redis
- **Add-ons marketplace** — 150+ дополнений (базы данных, мониторинг, email и др.)
- **Heroku CLI** — мощный командный интерфейс
- **Review Apps** — тестовые окружения для PR
- **Container Registry** — деплой Docker-образов

#### Тарифы и ценообразование

| Тип Dyno | Стоимость | Характеристики |
|----------|-----------|----------------|
| Eco | $5/мес | 512 MB RAM, для низкого трафика, засыпает при неактивности |
| Basic | $7/мес | Малые продакшн-приложения |
| Standard-1X | $25/мес | 1 GB RAM, не засыпает |
| Standard-2X | $50/мес | 2 GB RAM |
| Performance | от $250/мес | Высокопроизводительные конфигурации |

**Важно:** Бесплатный тариф Heroku был отменён в конце 2022 года.

#### Преимущества

- Проверенная временем платформа
- Огромная экосистема add-ons
- Отличная документация
- Простота использования
- Поддержка множества языков (Node.js, Ruby, Python, Java, PHP, Go, Scala, Clojure)
- Надёжность и стабильность

#### Недостатки

- Дороже современных конкурентов
- Нет постоянного бесплатного тарифа
- Ограниченные регионы (US, EU)
- Старый интерфейс
- «Платиновая» наценка за бренд

#### Идеально подходит для

- Enterprise-проектов с бюджетом
- Приложений, требующих специфические add-ons
- Команд с опытом работы на Heroku
- Ruby on Rails и Node.js-проектов

### DigitalOcean App Platform

![Type](https://img.shields.io/badge/type-Full--stack-blue)

## Примеры деплоя

### Render

**1. Подключить репозиторий в дашборде**
**2. Выбрать среду (Node.js/Python/Docker)**
**3. Настроить переменные окружения**
**4. Deploy!**

**render.yaml:**
```yaml
services:
  - type: web
    name: my-app
    env: node
    buildCommand: npm install && npm run build
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
```

**CLI:**
```bash
npx render-cli deploy
```

---

### Railway

**1. Войти через GitHub**
**2. Выбрать репозиторий**
**3. Добавить переменные окружения**
**4. Автоматический деплой**

**railway.json:**
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "npm start",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

**CLI:**
```bash
npm i -g @railway/cli
railway login
railway up
```

### Vercel

**1. Установить Vercel CLI**
**2. Выполнить `vercel login`**
**3. Выполнить `vercel` в папке проекта**
**4. Подтвердить деплой**

**vercel.json:**
```json
{
  "version": 2,
  "builds": [
    { "src": "package.json", "use": "@vercel/node" }
  ],
  "routes": [
    { "src": "/(.*)", "dest": "/" }
  ]
}
```

**CLI:**
```bash
npm i -g vercel
vercel login
vercel --prod
```

### Fly.io

**1. Установить Fly CLI**
**2. Выполнить `fly launch`**
**3. Выбрать регион и конфигурацию**
**4. Выполнить `fly deploy`**

**fly.toml:**
```toml
app = "my-app"
primary_region = "ams"

[build]
  dockerfile = "Dockerfile"

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = true
  auto_start_machines = true

[[vm]]
  cpu_kind = "shared"
  ram_mb = 256
```

**CLI:**
```bash
curl -L https://fly.io/install.sh | sh
fly auth login
fly launch
fly deploy
```

```dockerfile
FROM node:20-alpine

RUN npm ci --only=production

COPY . .

EXPOSE 3000

CMD ["node", "dist/main.js"]
```

```dockerfile
FROM python:3.11-slim

COPY requirements.txt .

COPY . .

EXPOSE 8000

CMD ["python", "main.py"]
```

## FAQ

**Q: Можно ли использовать несколько платформ одновременно?**

**A:** Да, популярная архитектура:
- **Vercel** (frontend) + **Supabase** (backend/БД) + **Cloudflare** (edge functions)
- **Netlify** (сайт) + **Railway** (API/бот) + **Supabase** (БД)

---

**Q: Как избежать vendor lock-in?**

**A:**
1. Использовать **Docker** для контейнеризации
2. Infrastructure as Code (**Terraform**, **Pulumi**)
3. Выбирать **self-hosted** решения (Coolify, CapRover)
4. Хранить конфигурацию в Git
5. Использовать стандартные БД (PostgreSQL, MySQL)

---

**Q: Что выбрать для production?**

**A:** Зависит от бюджета и требований:
- **Стартап с ограниченным бюджетом:** Railway, Render
- **Enterprise с требованиями к SLA:** Heroku, Northflank, Fly.io
- **РФ с 152-ФЗ:** Amvera, Timeweb Cloud, Coolify на российском VPS
- **High-load глобально:** Cloudflare Workers, Fly.io

---

**Q: Сколько стоит запустить Telegram-бота?**

**A:**
- **Railway:** ~$2-5/мес (бесплатно при <1000 пользователей)
- **Amvera:** ~200-500 руб/мес
- **Fly.io:** ~$3-5/мес
- **Coolify на VPS:** ~$5/мес (VPS) + 0 (ПО)

---

**Q: Нужен ли Docker для деплоя?**

**A:** Не обязательно, но рекомендуется:
- **Без Docker:** Vercel, Netlify, Render, Railway (автоопределение)
- **С Docker:** Любая платформа, полный контроль над средой

---

**Q: Как перенести проект с Heroku?**

**A:**
1. **На Render:**
   ```bash
   # Экспорт переменных окружения
   heroku config > .env
   
   # Подключить репозиторий в Render
   # Импортировать .env в Render Dashboard
   ```

2. **На Railway:**
   ```bash
   railway init
   railway variables set < .env
   railway up
   ```

3. **Экспорт PostgreSQL:**
   ```bash
   heroku pg:backups:capture
   heroku pg:backups:download
   pg_restore -U user -d database latest.dump
   ```

---

**Q: Что такое cold start и как избежать?**

**A:** Cold start — задержка при первом запуске serverless-функции после простоя.

**Платформы с cold start:**
- Vercel: ~100-500ms
- Cloudflare Workers: ~1-10ms (минимальный)
- Railway: ~500-2000ms

**Как избежать:**
1. Выбрать платформы без засыпания (Render платный, Heroku платный)
2. Использовать ping-сервисы (UptimeRobot, Cron-Job.org)
3. Выбрать Fly.io / Cloudflare (минимальный cold start)

## Миграция между платформами

### С Heroku на Render

1. Экспорт переменных окружения: `heroku config > .env`
2. Перенос PostgreSQL: `pg_dump` + `pg_restore`
3. Обновление DNS записей
4. Тестирование на домене

### С VPS на Coolify

1. Установка Coolify: `curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash`
2. Импорт Docker Compose через веб-интерфейс
3. Настройка доменов и SSL

### С Vercel на Netlify (и обратно)

1. Клонировать репозиторий
2. Подключить в новой платформе
3. Перенести переменные окружения
4. Обновить DNS

## Заключение

Выбор платформы для деплоя зависит от множества факторов: типа проекта, бюджета, требований к производительности, локализации данных и навыков команды. Современные PaaS-платформы значительно упрощают процесс развёртывания приложений, избавляя разработчиков от необходимости управлять инфраструктурой.

Для быстрого старта рекомендуется начать с платформ, предлагающих щедрые бесплатные тарифы (Vercel, Netlify, Railway, Cloudflare), а затем мигрировать на более мощные решения по мере роста проекта. Российские проекты с требованиями к локализации данных могут рассмотреть Amvera или Timeweb Cloud, а команды с навыками администрирования могут существенно сэкономить, выбрав self-hosted решения вроде Coolify.
