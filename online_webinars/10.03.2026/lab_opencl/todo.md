### 🔴 Приоритет 1: Производительность GPU

> **Проблема:** Сейчас GPU работает медленнее CPU из-за overhead на передачу данных

- [ ] **Zero-copy память**
  - Использовать `CL_MEM_ALLOC_HOST_PTR` для прямой передачи данных
  - Избежать лишнего копирования между хостом и устройством
  - **Ожидаемый эффект:** ускорение в 5-10x

- [ ] **Pinned memory (page-locked)**
  - Выделение памяти через `clCreateBuffer` с флагом `CL_MEM_ALLOC_HOST_PTR`
  - Ускорение передачи CPU↔GPU через PCIe
  - **Ожидаемый эффект:** ускорение передачи в 2-3x

- [ ] **Batch processing**
  - Группировка нескольких вычислений в один kernel call
  - Уменьшение overhead на запуск kernel
  - **Ожидаемый эффект:** ускорение в 2-5x для малых задач

- [ ] **Multiple command queues**
  - Асинхронные вычисления через несколько очередей команд
  - Перекрытие передачи данных и вычислений
  - **Ожидаемый эффект:** ускорение в 1.5-2x

## 📝 Заметки

### Сборка через CMake (рекомендуется)
```bash
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release \
         -DBUILD_TESTS=ON \
         -DBUILD_BENCHMARKS=ON \
         -DENABLE_PROFILING=ON
cmake --build . -j$(nproc)
ctest --verbose
```

### Сборка через Makefile
```bash
# Hash
cd hashing && make && ./hash

# Sieve
cd sieve && make && ./sieve

# Tests
cd tests && make run
```

### Запуск тестов
```bash
cd build
ctest --verbose
# или
ctest -R SHA256  # конкретный тест
```

### Профилирование
```bash
# Включить профилирование
export CL_PROFILE=1
./hash 100000 64 256

# NVIDIA Nsight Compute
ncu ./hash 100000 64 256

# Intel VTune
vtune -collect hotspots ./hash 100000 64 256
```

### Известные ограничения
- В CI/CD тесты выполняются только на CPU (нет GPU)
- Windows: требуется shlwapi.lib для PathRemoveFileSpecA
- GPU overhead значителен для малых объёмов данных (<10000 элементов)
