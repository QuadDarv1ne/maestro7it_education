## Алгоритмы

### 1. Решето Эратосфена

**CPU версия:**
Классический последовательный алгоритм:
1. Инициализация массива (все числа простые)
2. Для каждого простого p от 2 до √N:
   - Отметить все кратные p как составные (начиная с p²)
3. Подсчёт оставшихся простых чисел

**GPU версия:**
Параллельная реализация на OpenCL:
- Kernel `init_array`: инициализация массива на GPU
- Kernel `sieve_mark_multiples`: каждый поток отмечает кратные текущего простого
- Kernel `count_primes`: параллельный подсчёт с использованием атомиков

**Оптимизации:**
- Распределение работы между потоками (каждый поток обрабатывает часть кратных)
- Коалесцентный доступ к памяти
- Использование атомарных операций для подсчёта

### 2. Параллельное хэширование (SHA-256)

**CPU версия:**
Стандартная реализация SHA-256:
1. Padding сообщения до длины, кратной 512 битам
2. Разбиение на блоки по 512 бит
3. Для каждого блока: расширение до 64 слов и 64 раунда сжатия

**GPU версия:**
- Каждый поток вычисляет хэш для своего блока данных
- Все хэши вычисляются параллельно
- Поддержка сообщений разной длины

**Дополнительные хэши:**
- DJB2 — простой быстрый хэш
- FNV-1a — лучший для хэш-таблиц
- MurmurHash — быстрый с хорошим распределением

## Оптимизации

### Выбор размера work-group

Размер work-group (local_size) влияет на производительность:
- **Маленький (32-64)**: больше overhead, меньше параллелизм внутри группы
- **Средний (128-256)**: оптимально для большинства задач
- **Большой (512-1024)**: может быть ограничен hardware

Рекомендуется тестировать разные размеры для конкретного GPU.

### Рекомендации

1. **Для решета Эратосфена:**
   - Используйте local_size = 256 для оптимального баланса
   - Для больших N (>10^8) может потребоваться больше памяти

2. **Для хэширования:**
   - SHA-256: local_size = 256-512
   - Простые хэши (DJB2, FNV): можно использовать больший local_size

## Возможные ошибки

### "GPU не найден"
- Проверьте установку драйверов GPU
- Убедитесь, что OpenCL runtime установлен
- Программа автоматически переключится на CPU

### "Ошибка компиляции kernel"
- Проверьте версию OpenCL (требуется 1.2+)
- Убедитесь, что файл .cl находится в той же директории

### "Не удалось выделить память"
- Уменьшите параметр N или num_hashes
- Проверьте доступную память GPU

## Теоретические основы

### Решето Эратосфена
Алгоритм находит все простые числа до N за время O(n log log n).

Количество простых чисел до N приближённо равно N / ln(N) (теорема о распределении простых чисел).

### SHA-256
Криптографическая хэш-функция:
- Выход: 256 бит (32 байта)
- Размер блока: 512 бит
- Количество раундов: 64
- Стойкость к коллизиям: ~2^128 операций
