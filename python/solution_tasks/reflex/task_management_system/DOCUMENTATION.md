# Документация системы управления задачами

## Обзор

Система управления задачами - это веб-приложение, созданное с использованием фреймворка Reflex. Приложение предоставляет пользователю возможность создавать, отслеживать и управлять задачами с различными атрибутами.

## Архитектура приложения

### Состояние (State)

Класс `State` управляет состоянием всего приложения:

- `tasks`: список всех задач в системе
- `new_task_*`: переменные для хранения значений формы добавления новой задачи
- `filter_*`: переменные для фильтрации задач
- `task_counter`: счетчик для генерации уникальных ID задач

### Модель задачи (Task)

Класс `Task` представляет собой модель одной задачи с полями:

- `id`: уникальный идентификатор задачи
- `title`: название задачи
- `description`: описание задачи
- `priority`: приоритет задачи (Low, Normal, High, Critical)
- `status`: статус задачи (Pending, In Progress, Completed, Cancelled)
- `due_date`: дата выполнения
- `created_at`: дата создания
- `assigned_to`: имя исполнителя

### Функции управления

#### Добавление задачи
- `add_task()`: создает новую задачу на основе значений формы и добавляет в список

#### Удаление задачи
- `delete_task(task_id)`: удаляет задачу по ID

#### Изменение статуса
- `update_task_status(task_id, new_status)`: изменяет статус задачи
- `toggle_task_completion(task_id)`: переключает статус выполнения задачи

#### Фильтрация
- `get_filtered_tasks()`: возвращает отфильтрованные задачи
- `get_task_stats()`: возвращает статистику по задачам

## Компоненты интерфейса

### task_form()
Форма для добавления новой задачи с полями:
- Название задачи
- Описание
- Приоритет
- Срок выполнения
- Назначено

### task_card(task)
Отображает карточку задачи с:
- Названием и описанием
- Приоритетом и статусом
- Кнопками управления
- Информацией о назначении и сроке

### filter_controls()
Элементы управления для фильтрации задач по статусу и приоритету

## Технологии

- Reflex - фреймворк для создания веб-приложений на Python
- Python - основной язык программирования
- React - используется под капотом Reflex для отображения интерфейса
- TypeScript - для типизации (через Pydantic модели)

Приложение может быть расширено следующими возможностями:

- Совместное использование задач
